## 1. はじめに

### 1.1 本ツールの目的

CANoe 自動テストツールは、車載通信テストにおける以下の作業を自動化するためのツールです。

- **テストパターン作成**: CANdb（`.dbc`）/ LDF（`.ldf`）ファイルから信号情報を取得し、日本語でテストパターンを記述
- **テスト自動実行**: CANoe COM API を通じてテストパターンを自動実行
- **テスト結果判定**: 実行ログと期待値を比較し、OK/NG を自動判定
- **帳票出力**: テスト結果を Excel ファイルとして出力

### 1.2 動作環境

| 項目 | 要件 |
|------|------|
| OS | Windows 10 / 11 |
| Python | 3.10 以上 |
| CANoe | バージョン 10.0 以降（COM API 対応） |
| Azure OpenAI | GPT-4o デプロイメント（Phase 2 で使用） |

### 1.3 前提条件

- CANoe がインストール済みで、COM サーバーが有効化されていること
- テスト対象の CANdb（`.dbc`）または LDF（`.ldf`）ファイルが用意されていること
- Azure OpenAI を使用する場合、API キーとエンドポイントが取得済みであること

## 2. インストール

### 2.1 Python のインストール

1. [Python 公式サイト](https://www.python.org/downloads/) から Python 3.10 以上をダウンロード
2. インストーラーを実行し、**「Add Python to PATH」にチェック**を入れてインストール
3. コマンドプロンプトで確認:

```
python --version
```

`Python 3.10.x` 以上が表示されれば OK です。

### 2.2 ツールのインストール

```bash
# リポジトリをクローン
git clone https://github.com/chiguh28/canoe.git
cd canoe

# 仮想環境を作成・有効化
python -m venv .venv
.venv\Scripts\activate

# 本体をインストール（開発用依存含む）
pip install -e ".[dev]"
```

### 2.3 依存ライブラリのインストール

上記の `pip install -e ".[dev]"` で以下が自動的にインストールされます。

| ライブラリ | 用途 |
|-----------|------|
| cantools | CANdb（`.dbc`）ファイルのパース |
| ldfparser | LDF（`.ldf`）ファイルのパース |
| openpyxl | Excel 帳票生成 |
| httpx | Azure OpenAI API 通信 |

Windows 環境で CANoe 連携を使用する場合:

```bash
pip install -e ".[dev,windows]"
```

### 2.4 CANoe の設定確認

1. CANoe を起動
2. メニューから **「ツール」→「オプション」** を開く
3. **「一般」→「COM インターフェース」** が有効になっていることを確認
4. CANoe を閉じる（ツールから接続するため）

### 2.5 Azure OpenAI の設定

テストパターンの AI 変換（Phase 2）を使用する場合、以下の環境変数を設定してください。

```bash
set AZURE_OPENAI_ENDPOINT=https://your-resource.openai.azure.com/
set AZURE_OPENAI_API_KEY=your-api-key
set AZURE_OPENAI_DEPLOYMENT=your-deployment-name
```

永続的に設定する場合は、Windows の「システム環境変数」に追加してください。

## 4. 信号情報の読み込み（Phase 1）

### 4.1 DBC ファイルの読み込み

1. **「信号情報」タブ** を選択
2. **「ファイルを開く」ボタン** をクリック（またはメニュー「ファイル」→「ファイルを開く」）
3. ファイル選択ダイアログで `.dbc` ファイルを選択
4. 信号情報が一覧に表示される

読み込まれる情報:

| 列名 | 説明 | 例 |
|------|------|-----|
| 信号名 | CAN 信号の名称 | EngineSpeed |
| メッセージ名 | 所属メッセージの名称 | EngineData |
| メッセージ ID | CAN フレーム ID（16進数） | 0x100 |
| データ型 | signed / unsigned | unsigned |
| 最小値 | 物理値の下限 | 0.0 |
| 最大値 | 物理値の上限 | 8000.0 |
| 単位 | 物理単位 | rpm |
| プロトコル | 通信プロトコル | CAN |

### 4.2 LDF ファイルの読み込み

DBC ファイルと同じ操作手順で `.ldf` ファイルを読み込めます。LIN プロトコルの信号情報が表示されます。

### 4.3 複数ファイルの同時読み込み

ファイル選択ダイアログで複数のファイルを選択できます。DBC と LDF を混在して読み込むことも可能です。

読み込んだファイル名はツールバーに表示されます。

### 4.4 信号一覧の検索・ソート

**検索:**

1. 検索フィールドにキーワードを入力
2. 信号名またはメッセージ名に一致する信号がリアルタイムで絞り込まれる
3. 検索は大文字・小文字を区別しない

**ソート:**

各列のヘッダーをクリックすると、その列で昇順/降順にソートされます。同じ列を再度クリックするとソート順が反転します。

### 4.5 信号詳細の確認

信号一覧から信号を選択すると、以下の詳細情報を確認できます。

- 信号名・メッセージ名
- メッセージ ID（16進数表示）
- データ型・最小値・最大値・単位
- ノード情報（送信元 → 受信先）
- ソースファイル名
- プロトコル（CAN / LIN）

## 5. テストパターンの作成（Phase 2）

### 5.1 テストパターンの入力

**「テストパターン作成」タブ** で、テストケースを日本語で記述します。

各テストパターンには以下の情報を入力します:

| 項目 | 説明 | 入力例 |
|------|------|--------|
| テストケース名 | テスト内容の簡潔な説明 | エンジン回転数設定テスト |
| 対象信号 | テスト対象の信号名 | EngineData.EngineSpeed |
| 操作内容 | 実行する操作（日本語） | エンジン回転数を 2000rpm に設定する |
| 期待値 | 期待する結果（日本語） | スロットル開度が 20% 以上 |
| 前提条件 | テスト開始前の条件 | エンジン停止状態 |
| 待機時間 (ms) | 操作後の待機時間 | 1000 |
| 備考 | 追加情報 | — |

### 5.2 信号の選択

信号選択コンポーネントを使用して、Phase 1 で読み込んだ信号から対象信号を選択できます。

1. 検索フィールドにキーワードを入力
2. ドロップダウンに候補が表示される
3. 信号を選択すると詳細情報が表示される
4. **「追加」ボタン** で選択リストに追加

### 5.3 日本語によるテスト記述

操作内容と期待値は日本語で自由に記述できます。AI 変換で構造化データに自動変換されます。

**記述例:**

| 操作内容 | 期待値 |
|---------|--------|
| エンジン回転数を 2000rpm に設定する | スロットル開度が 20% 以上 |
| ブレーキペダルを 50% まで踏み込む | ブレーキランプが点灯 |
| 冷却水温度を監視する（操作なし） | 温度が 80℃〜100℃ の範囲内 |

### 5.4 一括変換（Azure OpenAI）

作成したテストパターンを Azure OpenAI（GPT-4o）で構造化データに一括変換します。

1. テストパターンを入力
2. **「一括変換」ボタン** をクリック
3. 変換の進捗が表示される
4. 完了するとプレビュー画面に結果が表示される

変換結果には以下のパラメータが含まれます:

| パラメータ | 説明 |
|-----------|------|
| signal_name | 操作対象の信号名 |
| message_name | メッセージ名 |
| action | 操作種別（set / get / wait） |
| value | 設定値 |
| channel | CAN チャネル |
| wait_ms | 待機時間（ミリ秒） |
| expected_value | 期待値 |
| tolerance | 許容誤差 |
| judgment_type | 判定タイプ（exact / range / change / timeout） |

### 5.5 プレビューと手動修正

変換結果はプレビュー画面で確認・修正できます。

1. 変換結果を確認
2. 必要に応じてパラメータを手動で修正
3. **「確定」ボタン** で確定

### 5.6 テストパターンの保存・読込

確定したテストパターンは JSON ファイルとして保存・読込できます。

**保存:**

テストパターンの確定後、JSON ファイルに出力されます。

**読込:**

保存済みの JSON ファイルを読み込んで、テスト実行に使用できます。

## 6. テストの実行（Phase 3）

### 6.1 CANoe 構成ファイルの設定

1. **「テスト実行」タブ** を選択
2. **「参照」ボタン** をクリックして CANoe 構成ファイル（`.cfg`）を選択
3. ファイルパスが表示されることを確認

### 6.2 テスト実行の開始

1. テストパターンが設定されていることを確認
2. **「実行開始」ボタン** をクリック
3. CANoe に自動接続し、テストが順次実行される

テスト実行の流れ:

```
テストパターン読込
    ↓
CANoe 接続
    ↓
構成ファイル読込
    ↓
測定開始
    ↓
各テストケースを順次実行:
  1. 前提条件の適用
  2. 信号値の設定（操作実行）
  3. 待機時間の経過
  4. 結果値の取得
    ↓
測定停止
    ↓
実行結果の表示
```

### 6.3 進捗の確認

テスト実行中は以下の情報が表示されます:

- **プログレスバー**: 全体の進捗率
- **カウンター**: 「N / 全件数」形式で進捗表示
- **現在のテスト名**: 実行中のテストケース名
- **ログ**: 実行ログがリアルタイムで表示

### 6.4 テストの中断

テスト実行中に **「中断」ボタン** をクリックすると、実行を中断できます。

- 実行中のテストケースの完了後に停止
- 未実行のテストケースは SKIPPED ステータスとなる
- 中断までの部分結果は保持される

### 6.5 実行ログの確認

テスト実行のログは以下の形式で保存されます:

- **CSV 形式**: `logs/TC-001_YYYYMMDD_HHMMSS.csv`
- **JSON 形式**: `logs/TC-001_YYYYMMDD_HHMMSS.json`

ログには信号の送受信履歴（タイムスタンプ、チャネル、メッセージ名、信号名、値、方向）が記録されます。

## 7. 結果の確認と帳票出力（Phase 4）

### 7.1 実行結果の確認

**「結果・帳票」タブ** でテスト実行結果を確認できます。

結果サマリには以下の情報が表示されます:

| 項目 | 説明 |
|------|------|
| 合計 | テストケース総数 |
| OK | 合格したテスト数 |
| NG | 不合格のテスト数 |
| ERROR | エラーが発生したテスト数 |
| SKIP | スキップされたテスト数 |
| 合格率 | OK / (OK + NG) × 100 |

### 7.2 判定結果の見方

本ツールは以下の 5 種類の判定タイプをサポートしています。

| 判定タイプ | 説明 | 判定条件 |
|-----------|------|---------|
| EXACT | 値一致 | \|実測値 − 期待値\| ≤ 許容誤差 |
| RANGE | 範囲内 | 最小値 ≤ 実測値 ≤ 最大値 |
| CHANGE | 値変化 | 信号値の変化方向（増加/減少/変化なし） |
| TIMEOUT | タイムアウト | 指定時間内に信号を受信 |
| COMPOUND | 複合条件 | 複数条件の AND / OR 組み合わせ |

### 7.3 Excel 帳票の出力

1. テスト実行が完了した状態で **「Excel 帳票出力」ボタン** をクリック
2. 保存先を選択
3. Excel ファイルが出力される

出力ファイル名の形式: `test_report_YYYYMMDD_HHMMSS.xlsx`

### 7.4 帳票の読み方

出力される Excel 帳票は以下の 3 シートで構成されます。

**サマリシート:**

| 列 | 内容 |
|-----|------|
| テストケース ID | TC-001 形式 |
| テスト名 | テストケースの名称 |
| 対象信号 | テスト対象の信号 |
| 期待値 | 期待する値 |
| 実測値 | 実際に取得した値 |
| 結果 | OK / NG / ERROR |
| 実行時間 | テストの実行時間 |

結果のセルは色分けされています:
- **OK**: 緑色
- **NG**: 赤色
- **ERROR**: 黄色

**統計シート:**

テスト全体の統計情報（合計、OK 数、NG 数、合格率、実行時間の合計など）が表示されます。

**詳細シート（テストケースごと）:**

各テストケースの詳細情報が個別のシートに記録されます。判定タイプ、対象信号、期待値、実測値、差分、判定理由が確認できます。

## 9. トラブルシューティング

### 9.1 ファイルが読み込めない

**症状:** DBC/LDF ファイルの読み込みでエラーが表示される

**対処法:**
1. ファイルが存在し、アクセス権限があることを確認
2. ファイルが空でないことを確認
3. ファイルの形式が正しい（`.dbc` または `.ldf`）ことを確認
4. CANoe で正常に読み込めるファイルであることを確認
5. ファイルの文字エンコーディングが正しいことを確認

### 9.2 Azure OpenAI に接続できない

**症状:** テストパターンの AI 変換でエラーが表示される

**対処法:**
1. 環境変数が正しく設定されていることを確認:
   ```bash
   echo %AZURE_OPENAI_ENDPOINT%
   echo %AZURE_OPENAI_API_KEY%
   echo %AZURE_OPENAI_DEPLOYMENT%
   ```
2. エンドポイント URL が正しいことを確認（末尾に `/` が必要）
3. API キーが有効で、期限切れでないことを確認
4. デプロイメント名が Azure ポータルの設定と一致していることを確認
5. ネットワーク接続を確認（プロキシ設定が必要な場合あり）

### 9.3 CANoe に接続できない

**症状:** テスト実行時に「CANoe に接続できません」エラー

**対処法:**
1. CANoe がインストールされていることを確認
2. CANoe の COM インターフェースが有効であることを確認
3. 他のアプリケーションが CANoe を使用していないことを確認
4. 管理者権限でツールを実行してみる
5. CANoe を一度起動してから閉じ、再度ツールを実行

### 9.4 テストが失敗する

**症状:** テストの判定結果が NG になる

**確認ポイント:**
1. テストパターンの信号名・メッセージ名が正しいか確認
2. 期待値と判定タイプが適切か確認
3. 待機時間が十分か確認（信号の応答時間を考慮）
4. CANoe の構成ファイル（`.cfg`）が正しいか確認
5. 実行ログ（CSV/JSON）で信号の送受信履歴を確認

### 9.5 帳票が出力できない

**症状:** Excel 帳票の出力でエラーが表示される

**対処法:**
1. 出力先のディレクトリが存在することを確認
2. 出力先に書き込み権限があることを確認
3. 同名のファイルが開かれていないことを確認
4. ディスクの空き容量を確認

## 10. FAQ

**Q: 対応している通信プロトコルは?**

A: CAN（`.dbc` ファイル）と LIN（`.ldf` ファイル）に対応しています。

**Q: CANoe のどのバージョンに対応していますか?**

A: CANoe バージョン 10.0 以降の COM API 対応バージョンで動作します。

**Q: Azure OpenAI なしでも使えますか?**

A: Phase 1（信号読み込み）は Azure OpenAI なしで使用できます。Phase 2 のテストパターン変換には Azure OpenAI が必要です。手動でパラメータを入力する場合は不要です。

**Q: Linux / macOS で使えますか?**

A: CANoe が Windows 専用のため、テスト実行（Phase 3）は Windows 環境が必要です。信号情報の読み込み（Phase 1）やテストパターンの作成（Phase 2）は Python が動作する環境であれば利用可能です。

**Q: テスト結果のログはどこに保存されますか?**

A: プロジェクトルート直下の `logs/` ディレクトリに CSV および JSON 形式で保存されます。ファイル名は `{テストケースID}_{日時}.csv` の形式です。

**Q: 一度に実行できるテストケース数に上限はありますか?**

A: ソフトウェア上の上限はありません。ただし、大量のテストケース（50 件以上）を実行する場合は、実行時間が長くなります。

**Q: テストパターンの JSON ファイル形式は?**

A: 以下の形式の JSON 配列です:

```json
[
  {
    "test_case_id": "TC-001",
    "test_case_name": "テスト名",
    "target_signal": "信号名",
    "operation": "操作内容",
    "expected_value": "期待値",
    "precondition": "前提条件",
    "wait_time_ms": 1000,
    "remarks": "備考"
  }
]
```

**Q: 判定タイプはどのように選択されますか?**
