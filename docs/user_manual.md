## はじめに

本ツールは、Vector 社製 CANoe を用いた車載通信テストにおいて、テストパターンの作成から実行、判定、帳票作成までを自動化する Python ツールです。

### 対応ファイル形式

| 形式 | 拡張子 | プロトコル | 説明 |
|------|--------|-----------|------|
| CANdb | `.dbc` | CAN | CAN バス信号定義ファイル |
| LDF | `.ldf` | LIN | LIN バス信号定義ファイル |

## インストール手順

### 動作要件

- Python 3.10 以上
- CANoe（Vector 社製、COM API 対応版）※ テスト実行時のみ必要
- Windows OS（CANoe COM API 使用時）

### 手順

1. リポジトリをクローン:

```bash
git clone https://github.com/chiguh28/canoe.git
cd canoe
```

2. パッケージをインストール:

```bash
pip install -e .
```

3. 開発用依存関係もインストールする場合:

```bash
pip install -e ".[dev]"
```

4. Windows 環境で CANoe COM API を使用する場合:

```bash
pip install -e ".[windows]"
```

5. Excel 帳票出力を使用する場合:

```bash
pip install -e ".[phase4]"
```

### インストール確認

```bash
python -c "import src; print('インストール成功')"
```

## 基本操作

### アプリケーションの起動

```bash
python -m src
```

### 画面構成

アプリケーションは以下の 4 つのタブで構成されています:

1. **信号情報タブ**: DBC/LDF ファイルの読み込みと信号一覧の表示
2. **テストパターン作成タブ**: テストパターンの作成と管理
3. **テスト実行タブ**: テストの実行制御と進捗表示
4. **結果・帳票タブ**: テスト結果の確認と Excel 帳票出力

### 信号情報の読み込み

1. 「信号情報」タブを選択
2. 「ファイルを開く」ボタンをクリック（または `Ctrl+O`）
3. DBC または LDF ファイルを選択
4. 信号一覧が Treeview に表示される

### 信号の検索・フィルタリング

- 検索ボックスに文字を入力すると、信号名・メッセージ名で絞り込み
- カラムヘッダーをクリックするとソート（昇順/降順切替）

## テストパターン作成

### 手動作成

1. 「テストパターン作成」タブを選択
2. 対象信号を選択
3. 操作内容（日本語で記述可能）を入力
4. 期待値を入力
5. 必要に応じて前提条件、待機時間を設定

### Azure OpenAI 変換

日本語のテスト記述を CANoe パラメータに自動変換できます:

1. テストパターンを作成
2. 「一括変換」を実行
3. プレビューで変換結果を確認
4. 必要に応じて手動修正
5. 確認して適用

### テストパターンの保存・読込

テストパターンは JSON 形式で保存・読込が可能です:

```json
[
  {
    "test_case_id": "TC-001",
    "test_case_name": "エンジン回転数確認",
    "target_signal": "EngineSpeed",
    "operation": "エンジン回転数を3000rpmに設定",
    "expected_value": "3000",
    "precondition": "エンジン稼働中",
    "wait_time_ms": 100,
    "remarks": ""
  }
]
```

## テスト実行

### 実行手順

1. 「テスト実行」タブを選択
2. CANoe 構成ファイル（`.cfg`）を選択
3. 「実行開始」ボタンをクリック（または `F5`）
4. 進捗バーで実行状況を確認
5. 必要に応じて「中断」ボタンで停止（または `Escape`）

### 実行フロー

各テストパターンに対して以下の手順で自動実行:

1. 前提条件の設定
2. 操作の実行（信号送信）
3. 指定時間の待機
4. 結果の取得と判定

### 判定方式

| 判定種別 | 説明 |
|---------|------|
| EXACT | 値一致判定（許容差指定可） |
| RANGE | 範囲判定（最小値〜最大値） |
| CHANGE | 変化検知（増加/減少/変化なし） |
| TIMEOUT | タイムアウト判定 |
| COMPOUND | 複合判定（AND/OR 組合せ） |

## 結果確認・帳票出力

### Excel 帳票

テスト結果を Excel ファイルとして出力します（`Ctrl+E`）:

- **サマリシート**: テスト全体の結果一覧（OK/NG 色分け表示）
- **統計シート**: 合格率、実行時間等の統計情報
- **詳細シート**: 各テストケースの詳細結果（テストケースごとに自動生成）

### JSON 出力

テスト結果・ログを JSON 形式でも保存できます:

- 実行結果 JSON: テストサマリと個別結果
- ログ JSON/CSV: テスト実行中の信号ログ

## Azure OpenAI 設定

日本語テストパターンの自動変換に Azure OpenAI を使用します。

### 環境変数の設定

```bash
export AZURE_OPENAI_ENDPOINT="https://your-endpoint.openai.azure.com/"
export AZURE_OPENAI_API_KEY="your-api-key"
export AZURE_OPENAI_DEPLOYMENT="your-deployment-name"
export AZURE_OPENAI_API_VERSION="2024-02-15-preview"  # オプション
```

### 機能

- 日本語の操作記述を CANoe COM API パラメータに自動変換
- バッチ変換対応（複数パターン一括処理）
- 変換結果のプレビュー・手動修正が可能
- リトライ機能（最大3回、指数バックオフ）
- レスポンスキャッシュによる効率化

## トラブルシューティング

### CANoe に接続できない

- **原因**: CANoe がインストールされていない、またはCOM API が利用できない
- **対処**: CANoe がインストールされ、COM API が有効であることを確認してください
- **対処**: `pip install pywin32` を実行して pywin32 をインストールしてください

### DBC/LDF ファイルが読み込めない

- **原因**: ファイルが破損している、またはエンコーディングが不正
- **対処**: ファイルが正しい形式であることを確認してください
- **対処**: UTF-8 エンコーディングであることを確認してください

### テスト実行が途中で停止する

- **原因**: CANoe との接続が切断された
- **対処**: CANoe が起動していることを確認し、再接続してください
- **対処**: 構成ファイル（.cfg）のパスが正しいことを確認してください

### Excel 帳票が出力できない

- **原因**: openpyxl がインストールされていない
- **対処**: `pip install openpyxl` を実行してください

### Azure OpenAI 変換が失敗する

- **原因**: 環境変数が設定されていない、またはAPIキーが無効
- **対処**: `AZURE_OPENAI_ENDPOINT`, `AZURE_OPENAI_API_KEY`, `AZURE_OPENAI_DEPLOYMENT` が正しく設定されていることを確認してください
